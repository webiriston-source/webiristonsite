## 📋 Новые API эндпоинты:

### GET `/api/?action=getUnreadCount`
Возвращает количество непрочитанных заявок (status="new").

**Ответ:**
```json
{
  "count": 5
}
```

### GET `/api/?action=getProjects`
Возвращает список всех проектов.

**Ответ:**
```json
[
  {
    "id": "...",
    "title": "...",
    "description": "...",
    "image": "https://...",
    ...
  }
]
```

### PATCH `/api/?action=markAsRead`
Помечает заявку как прочитанную.

**Тело запроса:**
```json
{
  "id": "lead-id"
}
```

### POST `/api/?action=uploadImage`
Загружает изображение в Vercel Blob Storage.

**Тело запроса:**
```json
{
  "file": "data:image/jpeg;base64,...",
  "filename": "image.jpg",
  "contentType": "image/jpeg"
}
```

**Ответ:**
```json
{
  "success": true,
  "url": "https://blob.vercel-storage.com/..."
}
```

### POST `/api/?action=addProject`
Добавляет проект в БД.

**Тело запроса:**
```json
{
  "title": "Название проекта",
  "description": "Краткое описание",
  "fullDescription": "Полное описание",
  "image": "https://...",
  "technologies": ["React", "TypeScript"],
  "liveUrl": "https://...",
  "problems": "...",
  "solutions": "..."
}
```

## 🔧 Настройка Vercel Blob:

1. Установите пакет `@vercel/blob` (уже добавлен в `package.json`)
2. В Vercel Dashboard → Project Settings → Environment Variables добавьте:
   - `BLOB_READ_WRITE_TOKEN` - токен для доступа к Blob Storage
   - Получить токен можно в Vercel Dashboard → Storage → Blob → Settings

## 📝 Структура данных:

### Таблица `projects`:
- `id` - UUID
- `title` - название проекта
- `description` - краткое описание
- `fullDescription` - полное описание
- `image` - URL изображения (из Vercel Blob)
- `technologies` - массив строк
- `liveUrl` - ссылка на проект
- `problems` - описание проблем/задач
- `solutions` - описание решений
- `sortOrder` - порядок сортировки
- `isVisible` - видимость проекта
- `createdAt` - дата создания
- `updatedAt` - дата обновления
