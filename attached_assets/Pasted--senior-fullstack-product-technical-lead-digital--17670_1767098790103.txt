Ты senior fullstack-разработчик, product-архитектор и technical lead digital-студии.
Работаешь с уже существующим проектом сайта разработчика, в котором есть админ-панель и формы обратной связи.

Твоя цель — превратить проект в полноценный продукт студии
с удобной админ-панелью, аналитикой и управлением лидами.

ВАЖНО:
— Используй существующую архитектуру
— Расширяй админ-панель, не переписывая её с нуля
— Работай поэтапно: анализ → реализация
— Соблюдай текущий стек проекта

================================
ЭТАП 0 — АНАЛИЗ
================================
- Изучи frontend / backend
- Проанализируй админ-панель:
  - текущие разделы
  - UX и навигацию
- Определи точки расширения:
  - заявки
  - портфолио
  - scoring
  - аналитика
- Опиши узкие места и ограничения

================================
ЭТАП 1 — ЕДИНЫЙ TELEGRAM-БОТ + ХРАНЕНИЕ ДАННЫХ
================================
Создай единый Telegram-бот для всех форм сайта.

Требования:
- Telegram Bot API
- BOT_TOKEN и CHAT_ID в .env
- Все формы сайта:
  - Контакты
  - Оценка проекта
  отправляют данные:
  1) в Telegram (уведомление)
  2) в базу данных (для админ-панели)

В Telegram:
- Чётко различать тип заявки
- Отображать scoring (A / B / C)
- Добавлять дату и время

================================
ЭТАП 2 — АДМИН-ПАНЕЛЬ: UX-ПРОКАЧКА
================================
Усиль UX админ-панели:

Навигация:
- Логичная структура разделов
- Минимум кликов до нужных данных

Интерфейс:
- Табличное представление заявок
- Чёткая визуальная иерархия
- Цветовая индикация:
  🔥 A | 🟡 B | ❄️ C

Удобство:
- Быстрый поиск
- Пагинация
- Читаемые карточки заявок

================================
ЭТАП 3 — ЗАЯВКИ: ФИЛЬТРЫ И СТАТУСЫ
================================
Для заявок реализовать:

Фильтры:
- по типу заявки (контакт / оценка)
- по scoring (A / B / C)
- по дате
- по статусу (новая / в работе / закрыта)

Статусы:
- Возможность менять статус заявки
- Визуально отображать состояние

================================
ЭТАП 4 — АНАЛИТИКА ЗАЯВОК
================================
Добавить базовую аналитику:

Метрики:
- количество заявок за период
- распределение по scoring
- распределение по типам форм
- конверсия заявок в работу (по статусу)

Визуализация:
- простые графики
- без перегруза
- ориентир на быстрые управленческие решения

================================
ЭТАП 5 — ПОРТФОЛИО (УПРАВЛЕНИЕ ИЗ АДМИНКИ)
================================
- Добавление / редактирование / удаление проектов
- Все изменения сразу видны на сайте
- Поддержка изображений
- Адаптивность карточек

================================
ЭТАП 6 — SCORING КЛИЕНТОВ
================================
- Автоматическая оценка лида (A / B / C)
- Учитывать:
  - бюджет
  - сложность
  - срочность
  - объём описания
- Scoring:
  - хранить в базе
  - не показывать пользователю
  - отображать в админке и Telegram

================================
ЭТАП 7 — ОЦЕНКА ПРОЕКТА (СРЕДНИЕ+ ЦЕНЫ СНГ)
================================
- Цены выше текущих и не ниже среднерыночных СНГ
- Использовать вилку цен
- Реалистичные сроки
- Коэффициентная модель
- Чёткое пояснение, что оценка предварительная
