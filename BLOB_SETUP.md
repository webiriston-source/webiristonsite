# Настройка Vercel Blob Storage

## 📋 Инструкция по получению токена BLOB_READ_WRITE_TOKEN

### Шаг 1: Создать Blob Storage (если еще не создан)

1. Откройте [Vercel Dashboard](https://vercel.com/dashboard)
2. Выберите ваш проект
3. Перейдите в раздел **Storage** (в боковом меню)
4. Если Blob Storage еще не создан:
   - Нажмите **Create Database**
   - Выберите **Blob**
   - Введите имя (например, `my-blob-storage`)
   - Выберите регион (ближайший к вашим пользователям)
   - Нажмите **Create**

### Шаг 2: Получить токен доступа

1. В разделе **Storage** → **Blob** → выберите ваше хранилище
2. Перейдите в **Settings** → **Tokens**
3. Нажмите **Create Token**
4. Настройки токена:
   - **Name**: `blob-read-write-token` (или любое другое имя)
   - **Permissions**: выберите **Read & Write**
   - **Expires**: можно оставить "Never" или установить срок действия
5. Нажмите **Create Token**
6. **ВАЖНО**: Скопируйте токен сразу - он показывается только один раз!

### Шаг 3: Добавить переменную окружения в Vercel

1. В Vercel Dashboard → ваш проект → **Settings** → **Environment Variables**
2. Нажмите **Add New**
3. Заполните:
   - **Key**: `BLOB_READ_WRITE_TOKEN`
   - **Value**: вставьте токен, скопированный на шаге 2
   - **Environment**: выберите нужные окружения:
     - ✅ Production
     - ✅ Preview
     - ✅ Development (опционально)
4. Нажмите **Save**

### Шаг 4: Передеплой проекта

После добавления переменной окружения нужно передеплоить проект:

1. В Vercel Dashboard → ваш проект → **Deployments**
2. Нажмите на последний деплой → **Redeploy**
3. Или сделайте новый коммит и пуш в GitHub (если настроен автодеплой)

## ✅ Проверка работы

После деплоя попробуйте загрузить изображение через форму портфолио. Если всё настроено правильно, изображение должно загрузиться и появиться в Blob Storage.

## 🔍 Отладка

Если загрузка не работает:

1. Проверьте логи в Vercel Dashboard → ваш проект → **Functions** → `api/index`
2. Убедитесь, что переменная `BLOB_READ_WRITE_TOKEN` добавлена и видна в нужных окружениях
3. Проверьте, что токен имеет права **Read & Write**
4. Убедитесь, что Blob Storage создан и активен

## 📝 Альтернатива: Использование переменной BLOB_READ_WRITE_TOKEN

Vercel Blob автоматически использует переменную окружения `BLOB_READ_WRITE_TOKEN`, если она установлена. Библиотека `@vercel/blob` автоматически подхватывает этот токен из окружения.
